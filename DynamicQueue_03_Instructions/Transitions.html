In each round, a random number of new tasks will become available to join the queue. This number will be drawn at random from a set of numbers {2,3,4}, with each number equally likely. We will refer to this random number as Number of New Tasks.

To determine the # of Tasks in the Queue in a given round, we will use the Number of New Tasks together with any Leftover Tasks from the previous round as follows:

  • In Round 1, there are no previous rounds and, therefore, # of Tasks in the Queue will be equal to the Number of New Tasks.
  • In Round >1, # of Tasks in the Queue will be determined in two steps:
    • First, we will determine the number of Leftover Tasks from the previous round.
      • If # of Tasks in the Queue in the previous round is less than or equal to the Total Capacity in the previous round, then the number of Leftover Tasks is 0.
      • If # of Tasks in the Queue in the previous round is greater than the Total Capacity in the previous round, then the number of Leftover Tasks is (# of Tasks in the Queue − Total Capacity).
    • Second, we will determine the # of Tasks in the Queue in the current round by adding the Leftover Tasks to the Number of New Tasks in the current round.
  • Importantly, the Queue is capped at 4 tasks. That is, any tasks that arrive beyond the limit of 4 will be discarded. For example, if in a given round there are 2 leftover tasks in the queue and 3 new tasks arrive that round, then the queue will contain 4 tasks and 1 will be discarded.
